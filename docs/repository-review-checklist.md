# Repository Review Checklist

Use this checklist before editing analysis code in an existing project.

## A. Problem and Intent

- [ ] I can state the main project question in one sentence.
- [ ] I understand who the end user is and what decision they need to make.
- [ ] I can explain why this project exists beyond model accuracy.

- [ ] The README explains the problem clearly.
- [ ] Dataset sources are named and linked.
- [ ] Workflow is documented from data to insight.
- [ ] Key outputs or expected deliverables are visible.
- [ ] Basic run or exploration instructions are included.

- [ ] I can distinguish exploratory artifacts from finalized outputs.
